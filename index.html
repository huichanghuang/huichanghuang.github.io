Hexo + Github Pages 搭建个人博客

Posted on 2023-08-17

Hexo是一款基于Node.js的静态博客框架，依赖少易于安装使用，支持 GitHub Flavored Markdown 的所有功能，同时可以方便生成静态网页托管在GitHub和Heroku上，是搭建博客的首选框架。

1. 前置工作如下，比较简单，不多赘述

  1. 注册github账号, github官网
  2. 安装git, git官网
  3. 安装node.js, node.js官网
  4. 本地新建一个文件夹存放博客源文件

2. 创建github远程托管仓库

  1. 到github个人页面点击Repositories，之后再点击New新建一个仓库

    出现以下页面，在Repository name填入对应的信息，必须是自己的github用户名.github.io的格式，然后勾选初始化Add a README file，最后点击右下角Create repository。
  2. 创建完成后，在浏览器输出https://XXX.github.io/ （XXX为自己的github用户名）会出来一个非常简陋的网站，类似这样。因为我们后续主要使用hexo搭建，暂时可以不用管网页主题配置啥的。

3. 安装hexo

  1. 打开新建的博客源文件夹，鼠标右键打开git bash here
  2. 打开git的控制台窗口后，输入npm install -g hexo-cli 安装hexo

4. 初步生成本地网站配置

  1. 在打开的博客源文件路径下的git控制台窗口输入‘hexo init`初始化本地网站配置

3. 授权远程服务器（or本地）连接github（前提是安装了git和ssh）

  1. 配置github用户名和邮箱
  2. rsa方式连接
    1. ssh-keygen创建密钥
    2. 将~/.ssh/id_rsa.pub的内容添加到github创建新的key
    3. ssh -T git@github.com连接测试
    4. 出现Hi xxx! You've successfully authenticated, but GitHub does not provide shell access.则表示连接成功
  3. ecdsa方式连接
    1. ssh-keygen -t ecdsa -b 521 -C "myname@mymail.com"创建密钥
    2. 将~/.ssh/id_ecdsa.pub的内容添加到[github]创建新的key
    3. ssh -T git@github.com连接测试
    4. 出现Hi xxx! You've successfully authenticated, but GitHub does not provide shell access.则表示连接成功
